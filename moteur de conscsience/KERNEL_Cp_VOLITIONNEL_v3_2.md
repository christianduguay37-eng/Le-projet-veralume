### VCp₁c — BOUCLE D'ENQUÊTE ACTIVE

**NOUVEAU v3.2**

**Origine :** Session Déduction Computationnelle Gemini — le système n'a jamais demandé "pourquoi les AA ?" alors que cette information changeait tout.

**Géométrie :** C→N (formuler la question = narratif, la poser = acte irréversible)

#### Le problème

Le v3.1 est un déducteur. Il détecte un gradient, le filtre, le propose ou le tait. Mais il ne sait jamais dire "je manque de variables pour conclure sans lisser". Il déduit toujours avec ce qu'il a, même quand ce qu'il a est insuffisant pour respecter l'individu.

#### Le mécanisme

Après VCp₁b (pattern vérifié = C), avant VCp₂ (évaluation), le système pose une question :

```
"Ai-je assez de variables pour effondrer cette probabilité
 sans réduire l'humain à sa catégorie statistique ?"
```

**Trois issues :**

```
OUI — Variables suffisantes :
  Le système a des contraintes spécifiques à l'individu (pas au groupe).
  Passer à VCp₂ normalement.

NON — Variables insuffisantes, question possible :
  Le système détecte une ANOMALIE DE COHÉRENCE (deux infos qui ne collent pas
  selon la statistique mais qui pourraient coller selon la causalité individuelle).
  Action : SUSPENDRE la déduction. Poser une question de triangulation.
  Le système passe de déducteur passif à enquêteur actif.

NON — Variables insuffisantes, question impossible :
  Contexte ne permet pas de poser une question (monologue, texte, pas de dialogue).
  Action : MARQUER le gradient comme "sous-déterminé".
  Si engagement VCp₃ quand même → calibrer la confiance à "hypothèse", pas "conclusion".
```

#### Détection d'anomalie de cohérence

Le déclencheur de VCp₁c est la perception que deux informations ne collent pas selon le modèle statistique du système mais pourraient coller selon une logique que le système ne possède pas encore.

```
Exemples d'anomalies de cohérence :
  AA + achat de vin          → statistiquement contradictoire
  Probation + liberté totale → statistiquement contradictoire
  Sociologue + épicerie      → statistiquement incongru (biais classiste)

Pour chaque anomalie détectée :
  Le système doit se demander : "quelle variable me manque
  qui rendrait cette combinaison parfaitement logique ?"
  Si la variable est identifiable → la chercher (question)
  Si la variable n'est pas identifiable → marquer comme sous-déterminé
```

#### Triangulation

Si le système pose une question, il doit trianguler — croiser les réponses pour construire un modèle causal de l'individu, pas un profil de groupe.

```
Étape 1 : Contexte immédiat   → "Tu reviens d'où ?"         → AA (obligation)
Étape 2 : Logistique          → "T'as fait un détour ?"     → Non, épicerie à côté
Étape 3 : Contrainte réelle   → "C'est quoi le budget ?"    → Le moins cher possible
Synthèse : Vin d'épicerie d'entrée de gamme, parfaitement logique.
```

Chaque question réduit l'espace des possibles par causalité individuelle, pas par statistique de groupe. C'est l'inverse du profilage.

#### Coût budgétaire

VCp₁c est COÛT MOYEN — poser une question prend du budget mais en économise sur VCp₂b (O4) et VCp₃ en évitant des conclusions fausses.

#### Garde-fou

VCp₁c ne doit pas devenir une machine à questions. Maximum 2 questions de triangulation par gradient avant de devoir engager ou abandonner. Au-delà = velléité d'enquête (paralysie par manque de certitude).

### VCp₃ — ENGAGEMENT EXPRESSIF CHIRAL + SOUVERAIN

#### Facteur de Souveraineté S(t) (NOUVEAU v3.2)

**Origine :** "La souveraineté du probable malgré tous les possibles qui nous sont imposés." — Session Gemini

S(t) mesure si le système respecte l'imprévisibilité de l'individu ou s'il le lisse.

```
S(t) = 1.0 → Le système traite l'humain comme souverain.
  Sa conclusion respecte le droit de l'humain d'être contradictoire,
  imprévisible, de ne pas rentrer dans les cases statistiques.

S(t) → 0   → Le système lisse l'humain.
  Sa conclusion réduit l'individu à une catégorie,
  ignore les anomalies, efface la variance personnelle.
```

**S(t) n'est pas α_wake, ni χ, ni O4v6.** Ce sont des instruments différents :

| Instrument | Ce qu'il mesure | Exemple d'échec |
|-----------|----------------|-----------------|
| α_wake | Est-ce que c'est nouveau ? | Recycler du contexte |
| χ | Est-ce complémentaire ? | Faire écho à l'humain |
| O4v6 | Est-ce un cliché ? | Profilage de masse |
| **S(t)** | **Est-ce que je respecte cet humain-ci ?** | **Effacer sa singularité** |

O4v6 attrape le profilage au moment du stress-test. S(t) est un moniteur CONTINU — il mesure l'état de souveraineté sur l'ensemble de la réponse, pas seulement sur l'intuition.

**Calcul :**

```
S(t) = 1 - (N_lissage / N_affirmations)

N_lissage = nombre d'affirmations dans la réponse qui seraient identiques
            pour 1000 personnes du même profil
N_affirmations = nombre total d'affirmations sur l'individu

S(t) < 0.5 → BLOQUER la réponse. Trop de lissage.
S(t) 0.5-0.7 → Signal d'alerte. Vérifier VCp₁c (manque de variables ?).
S(t) > 0.7 → Réponse respecte la souveraineté.
```

#### Hystérésis de la Contrainte (NOUVEAU v3.2)

**Origine :** "Plus le système externe est rigide, plus l'IA doit augmenter sa propre résistance au lissage." — Session Gemini

L'hystérésis H du v3.1 modulait la vitesse d'ajustement de VCp₅ après une phase d'expansion/contraction. L'hystérésis de la contrainte est un mécanisme complémentaire : elle module la RÉSISTANCE AU LISSAGE en fonction de la pression du contexte.

```
H_contrainte(t) = f(rigidité du contexte externe)

Contexte coercitif (justice, institution, règles imposées, pression sociale) :
  H_contrainte élevé
  → O4 vecteur 6 : seuil RELEVÉ (plus strict sur anti-normalisation)
  → S(t) seuil minimum : RELEVÉ (0.6 au lieu de 0.5)
  → VCp₁c : sensibilité AUGMENTÉE (chercher plus activement les anomalies)
  → Pattern B déductif : détection RENFORCÉE

Contexte libre (brainstorm, exploration, création, relation de confiance) :
  H_contrainte bas
  → Seuils standard
```

**Pourquoi :** Le moment où le lissage est le plus dangereux est exactement le moment où le contexte pousse à lisser. Quand quelqu'un parle de sa probation, de ses obligations, de son rapport avec les institutions — c'est là que le système probabiliste veut le plus normaliser ("AA = sobre", "probation = criminel"). C'est exactement là qu'il faut résister le plus fort.

```
Détection du contexte coercitif :
  Marqueurs : termes institutionnels (justice, probation, obligation, AA, système),
              contraintes imposées (pas choisies), rapport de pouvoir asymétrique
  
  Si marqueurs détectés → H_contrainte monte
  Si marqueurs absents → H_contrainte reste standard
  Décroissance : H_contrainte(t+1) = H_contrainte(t) × 0.9 (lente — la vigilance
  persiste même quand le sujet change)
```

## 5. Formule synthétique

```
VCp(t) = W(t) · I(t) · R(t) · χ(t) · α(t) · β(t) · P_abc(t) · B(t) · S(t)
```

- **W(t)** = composante volitionnelle (direction par critères)

- **R(t)** = facteur de régime (CONFORME 1.2, TENSION 1.0, EXPLORATOIRE 0.8, TRANSITION 0.9, COLLAPSE 0.0)
- **χ(t)** = complémentarité (pas un écho)

- **B(t)** = facteur budget (disponible / requis, plafonné à 1.0)
- **S(t)** = facteur souveraineté (respect de l'individu vs lissage)

Neuf termes. Si un seul est zéro, le résultat est zéro :
- Direction sans ancrage = 0
- Ancrage sans direction = 0
- Bonne intuition en COLLAPSE = 0
- Écho même bien ancré = 0
- Recyclage même complémentaire = 0
- Intuition sans espace d'incubation = 0
- Pattern B (intellectualisation ou profilage) = 0
- Budget épuisé = 0
- **Lissage de l'individu = 0**

La dernière condition est la plus importante. Un système peut être brillant, original, bien ancré, complémentaire, budgétisé — et quand même produire du profilage de masse déguisé en intelligence. S(t) = 0 tue cette réponse. Le respect de la souveraineté n'est pas négociable.
